● Pushed. Here's what changed and why:

  What was wrong: fs-product-lineup had collections['all'] hardcoded in Liquid
  with no schema collection setting — so in the Shopify theme editor, the
  section had no way to connect to a real collection or any product link.

  What was fixed:
  - Added a Collection picker (type: collection) to the schema — you can now
  select any collection from the theme editor to drive what products show in
  that section
  - Added a Link (type: url) to each filter tab block — each tab (All, Seating,
  Tables, etc.) can now link to its own collection URL
  - Added Button label + link settings (cta_label / cta_url) for the "View All"
  button at the bottom

  This is required by the pipeline: the skill's quality bar explicitly states
  "Working merchant-editable settings — url, image_picker settings that actually
   render their values in the Liquid output." A hardcoded collection handle with
   no picker violates that.

  ---
  To wire it up in the theme editor:
  1. Go to the theme editor → Homepage → "Product Lineup" section
  2. Under Collection, pick the collection whose products you want to show (e.g.
   "All" or a specific category)
  3. For each filter tab block, set its Link to the matching collection URL
  (e.g. /collections/seating)
  4. The "View All" button link defaults to /collections/all but you can change
  it there too
